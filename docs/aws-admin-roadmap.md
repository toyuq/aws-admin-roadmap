# AWS Admin 입문 로드맵

이 문서는 AWS Admin 역할을 처음 맡는 사람이 무엇을 먼저 배우고, 어떤 순서로 실무 감각을 쌓아가면 좋은지 정리한 입문 로드맵입니다.

## 목표

- AWS의 기본 구조를 이해한다.
- IAM, 네트워크, 배포, 모니터링, 비용 관리를 안전하게 다룬다.
- 클릭 중심이 아니라 기록과 자동화 중심으로 운영 습관을 만든다.

## 단계별 로드맵

### 1단계: AWS 기본 개념 익히기

먼저 AWS가 어떤 서비스 묶음인지 감을 잡습니다.

- EC2: 서버
- S3: 저장소
- RDS: 관리형 데이터베이스
- CloudWatch: 로그, 지표, 알람
- Route 53: DNS
- VPC: 네트워크 경계

기록 포인트:

- 오늘 배운 서비스 3개
- 각 서비스의 역할 1줄 요약
- 헷갈린 용어

### 2단계: IAM과 보안 이해하기

AWS Admin에서 가장 중요한 축입니다.

- 사용자(User), 역할(Role), 정책(Policy)의 차이
- 최소 권한 원칙
- MFA 적용
- Access Key 관리
- Secrets Manager / Parameter Store 개념

기록 포인트:

- 내가 접근 가능한 계정/역할 목록
- 실수하기 쉬운 권한 설정
- 확인이 필요한 보안 항목

### 3단계: 네트워크 기초 익히기

AWS는 네트워크를 모르면 운영이 막히기 쉽습니다.

- 퍼블릭/프라이빗 서브넷
- 보안 그룹(Security Group)
- NACL
- Internet Gateway / NAT Gateway
- 라우팅 테이블

기록 포인트:

- 현재 서비스가 퍼블릭인지 프라이빗인지
- 인바운드/아웃바운드 규칙
- 장애가 났을 때 확인할 네트워크 항목

### 4단계: 배포와 운영 흐름 익히기

서비스를 올리는 것보다 안정적으로 바꾸는 것이 더 중요합니다.

- 배포 파이프라인 이해
- 롤백 방법
- 로그 확인 방법
- 알람 확인 및 대응 순서
- 백업/복구 흐름

기록 포인트:

- 배포 순서
- 롤백 기준
- 장애 시 첫 10분 행동

### 5단계: 비용 관리 익히기

운영자는 성능만큼 비용도 봐야 합니다.

- 비용이 어디서 발생하는지 확인
- 태그 전략
- 불필요 리소스 정리
- 예산 알람
- 스냅샷, 저장소, 트래픽 비용 확인

기록 포인트:

- 이번 주 비용 증가 요인
- 정리 가능한 리소스
- 비용 알람 설정 상태

### 6단계: IaC와 자동화 도입하기

클릭보다 코드로 관리하는 습관을 만듭니다.

- Terraform 또는 CloudFormation 또는 CDK
- 반복 작업 자동화
- 변경 이력 남기기
- 운영 문서와 코드 연결

기록 포인트:

- 자동화할 수 있는 작업 목록
- 수동 작업에서 코드로 옮길 항목
- 리뷰가 필요한 인프라 변경

## 권장 학습 순서

1. AWS 핵심 서비스 개념
2. IAM과 보안
3. VPC와 네트워크
4. 배포/로그/알람
5. 비용 관리
6. IaC와 자동화

## 실무 마인드

- 최소 권한을 우선한다.
- 바꾸기 전에 되돌릴 방법을 생각한다.
- 기록 없이 작업하지 않는다.
- 모르면 추측하지 말고 로그와 설정을 먼저 본다.
- 반복 작업은 자동화한다.
