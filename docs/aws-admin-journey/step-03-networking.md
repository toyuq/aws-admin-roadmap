# Step 03. 네트워크 기초

## 목표

- AWS 네트워크 구조를 설명할 수 있다.
- 퍼블릭/프라이빗 환경 차이를 이해한다.

## 복습용 정리

### 1. AWS 네트워크를 이해하는 큰 그림

- AWS 네트워크는 단순히 서버가 있는 것이 아니라, **어떤 네트워크 안에 어떤 자원이 있고 어떻게 연결되는지**를 이해하는 것이 중요하다.
- 그 중심이 되는 개념이 **VPC**이다.
- VPC 안에 서브넷, 라우팅, 보안 설정이 함께 구성된다.

### 2. VPC

- **VPC (Virtual Private Cloud)** 는 AWS 안에서 독립적으로 구성하는 가상 네트워크 공간이다.
- 한 서비스나 프로젝트에 필요한 네트워크 환경을 분리해서 만들 수 있다.
- 외부와 직접 연결할지, 내부 전용으로 둘지 등을 설계할 수 있다.

### 3. Subnet

- Subnet은 VPC를 더 작은 네트워크 영역으로 나눈 것이다.
- 보통 **퍼블릭 서브넷**과 **프라이빗 서브넷**으로 나눠 생각한다.
- 어떤 서브넷에 배치되느냐에 따라 외부 접근 가능 여부가 달라진다.
- Subnet은 IP 주소 범위로도 표현되며, 이때 **CIDR 표기**를 함께 사용한다.

### 3-1. 서브넷 마스크와 CIDR

- **서브넷 마스크(subnet mask)** 는 IP 주소에서 네트워크 영역과 호스트 영역을 나누는 기준이다.
- 예를 들어 `255.255.255.0` 같은 값이 서브넷 마스크다.
- AWS에서는 서브넷을 만들 때 보통 서브넷 마스크를 직접 쓰기보다 **CIDR 표기**를 사용한다.
- CIDR은 `10.0.1.0/24`처럼 `주소/비트수` 형태로 쓴다.

### 3-2. CIDR 표기 읽는 법

- `10.0.1.0/24`는 `10.0.1.0`부터 시작하는 네트워크 범위를 뜻한다.
- `/24`는 앞의 24비트가 네트워크 부분이라는 의미다.
- 남은 8비트는 호스트 부분이므로, 그 범위 안에서 사용할 수 있는 IP가 정해진다.
- 쉽게 말하면 `/숫자`가 클수록 사용할 수 있는 IP 개수는 줄어든다.
- 예시:
	- `/24` = 비교적 작은 네트워크 범위
	- `/16` = 더 큰 네트워크 범위

### 3-3. AWS Subnet과의 관계

- AWS에서 말하는 Subnet은 단순한 IP 범위만이 아니라, VPC 안의 실제 네트워크 구역이다.
- 하지만 그 구역의 크기와 범위는 CIDR로 정한다.
- 예를 들어 `10.0.1.0/24` 같은 값을 서브넷에 지정하면, 해당 범위 안의 IP를 사용할 수 있다.
- 따라서 AWS Subnet을 이해할 때는 **네트워크 구역**과 **IP 범위(CIDR)** 를 같이 봐야 한다.

### 3-4. 네트워크 주소와 브로드캐스트 주소

- `10.0.1.0`은 **네트워크 주소**다.
	- 해당 서브넷 자체를 대표하는 주소다.
	- 보통 서버에 직접 할당하지 않는다.
- `10.0.1.255`는 **브로드캐스트 주소**다.
	- 해당 서브넷 안의 모든 장비에 한 번에 알릴 때 쓰는 특별한 주소다.
	- 보통 일반 호스트 IP로 사용하지 않는다.
- `10.0.1.0/24`에서는 보통 `10.0.1.1`부터 `10.0.1.254`까지를 실제 사용 가능한 IP로 본다.
- 즉, 네트워크 주소는 시작점 역할, 브로드캐스트 주소는 전체 알림용 역할이라고 이해하면 쉽다.

### 4. 퍼블릭 서브넷과 프라이빗 서브넷

- **퍼블릭 서브넷**: 인터넷과 연결될 수 있는 서브넷
- **프라이빗 서브넷**: 인터넷에서 직접 접근하지 않는 내부 전용 서브넷
- 웹 서버는 퍼블릭 서브넷에 두고, DB는 프라이빗 서브넷에 두는 식으로 분리하는 경우가 많다.

### 5. Internet Gateway

- Internet Gateway는 VPC와 인터넷을 연결하는 통로다.
- 퍼블릭 서브넷의 자원이 외부와 통신하려면 이 연결이 필요하다.
- 인터넷에 노출해야 하는 리소스가 있을 때 사용한다.

### 6. NAT Gateway

- NAT Gateway는 프라이빗 서브넷의 자원이 **외부로 나가는 통신**을 할 수 있게 해주는 장치다.
- 외부에서 프라이빗 서브넷으로 직접 들어오게 하지는 않는다.
- 보안상 내부 자원을 숨기면서도 업데이트나 패키지 다운로드 같은 아웃바운드 통신이 필요할 때 사용한다.

### 7. Security Group

- Security Group은 인스턴스 단위의 방화벽처럼 동작한다.
- 어떤 IP에서 어떤 포트로 들어올 수 있는지 규칙을 정한다.
- 보통 **허용 규칙 중심**으로 생각하면 된다.

### 8. NACL

- NACL(Network ACL)은 서브넷 단위의 추가 보안 규칙이다.
- 서브넷 전체에 대해 트래픽을 제어한다.
- Security Group보다 더 넓은 범위에서 적용되며, 허용과 거부를 함께 다룰 수 있다.

### 9. 라우팅 흐름

- 라우팅은 "이 트래픽을 어디로 보낼 것인가"를 정하는 것이다.
- VPC 안에서 패킷이 어디로 가는지 결정하는 역할을 한다.
- 인터넷, NAT, 다른 네트워크 연결 여부는 라우팅 설정과 연결된다.

### 10. 구성 예시 감각

- **웹 서버**: 퍼블릭 서브넷 + Internet Gateway + Security Group
- **DB 서버**: 프라이빗 서브넷 + Security Group
- **내부 서버 업데이트**: 프라이빗 서브넷 + NAT Gateway

### 10-1. 웹서버 ↔ WAS 서버 연결 흐름 예시

- 사용자가 브라우저에서 웹사이트에 접속하면, 먼저 **웹서버(퍼블릭 서브넷)** 로 요청이 들어온다.
- 웹서버는 필요한 경우 **프라이빗 서브넷의 WAS 서버** 로 내부 요청을 보낸다.
- 이때 중요한 것은 **Internet Gateway가 아니라 라우팅과 보안 설정**이다.
- 확인할 항목은 보통 다음과 같다.
	- 웹서버와 WAS 서버가 같은 VPC 안에 있는지
	- 라우트 테이블이 내부 통신을 막고 있지 않은지
	- WAS 서버의 Security Group이 웹서버의 요청을 허용하는지
	- NACL이 서브넷 단위에서 트래픽을 차단하지 않는지
- 정리하면, **IGW는 웹서버가 인터넷과 통신할 때 필요하고, 웹서버↔WAS 서버 내부 연결은 주로 Route Table과 Security Group을 확인한다.**

### 10-2. WAS / 인스턴스 / 컨테이너 비교

- **WAS (Web Application Server)**: 웹 요청을 받아서 비즈니스 로직을 처리하는 서버 역할이다.
	- 예: Tomcat, WebLogic, JBoss
- **인스턴스(instance)**: 실제로 동작하는 서버 1대 또는 실행 단위를 말한다.
	- AWS에서는 보통 EC2 인스턴스를 의미하며, 가상 서버 1대로 이해하면 쉽다.
- **컨테이너(container)**: 애플리케이션과 실행 환경을 묶어서 어디서나 비슷하게 실행되도록 만든 단위다.
	- 예: Docker 컨테이너 안에서 Tomcat 실행
- 관계를 쉽게 보면 다음과 같다.
	- **인스턴스**는 서버가 올라가는 실행 환경
	- **WAS**는 그 인스턴스 위에서 동작하는 애플리케이션 서버 역할
	- **컨테이너**는 WAS 같은 애플리케이션을 담아 실행하는 방식
- 한 줄로 정리하면, **WAS는 역할, 인스턴스는 서버 1대, 컨테이너는 애플리케이션 실행 단위**라고 이해하면 된다.

### 11. 자주 헷갈리는 점

- **VPC와 Subnet**: VPC는 큰 네트워크 공간, Subnet은 그 안의 작은 구역이다.
- **Security Group과 NACL**: Security Group은 인스턴스 중심, NACL은 서브넷 중심이다.
- **Internet Gateway와 NAT Gateway**: IGW는 인터넷 직접 연결, NAT는 프라이빗 서브넷의 외부 통신용이다.
- **퍼블릭 서브넷과 퍼블릭 IP**: 퍼블릭 서브넷이라고 해서 무조건 외부 노출되는 것은 아니며, 라우팅과 보안 설정도 함께 봐야 한다.

### 12. 스스로 답해볼 질문

- VPC는 왜 필요한가?
- Subnet은 VPC 안에서 어떤 역할을 하는가?
- 퍼블릭 서브넷과 프라이빗 서브넷은 어떻게 다른가?
- Internet Gateway는 언제 필요한가?
- NAT Gateway는 왜 쓰는가?
- Security Group과 NACL은 어떻게 다른가?
- 웹 서버와 DB 서버를 왜 다른 서브넷으로 나누는가?

### 13. 암기용 짧은 정리

- VPC = 가상 네트워크 공간
- Subnet = VPC 안의 작은 네트워크 구역
- Subnet mask = 네트워크와 호스트를 나누는 기준
- CIDR = IP 범위를 `/숫자`로 표현하는 방식
- Public Subnet = 인터넷 연결 가능
- Private Subnet = 내부 전용
- IGW = 인터넷 연결 통로
- NAT Gateway = 프라이빗 서브넷의 외부 통신 통로
- Security Group = 인스턴스 방화벽
- NACL = 서브넷 방화벽
